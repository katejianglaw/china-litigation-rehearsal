# Hearing Playbook

Use this reference after one or more simulations, or when the user asks for a final pre-hearing manual.

## Goal

Produce a compact, practical document the lawyer can review before the hearing. It should convert rehearsal outputs into decisions, scripts, evidence actions, and checklists.

## Prioritization

Always divide actions by timing:

| 优先级 | 事项 | 负责人 | 截止时间 | 影响 | 状态 |
| --- | --- | --- | --- | --- | --- |
| P0 | 开庭前必须完成 |  |  |  |  |
| P1 | 尽量完成 |  |  |  |  |
| P2 | 备选准备 |  |  |  |  |

P0 should include only items that materially affect outcome, admissibility, proof clarity, or courtroom performance.

## Expression Scripts

When drafting oral scripts:

- conclusion first;
- facts before doctrine unless the issue is purely legal;
- cite evidence numbers where available;
- avoid adjectives that the evidence cannot support;
- preserve fallback positions;
- do not concede facts or legal effects casually.

Use:

```text
30秒版：一句结论 + 两个证据锚点 + 一个请求。
60秒版：结论 + 事实链 + 证据链 + 法律后果。
3分钟版：争议焦点 + 我方证明路径 + 对方抗辩回应 + 请求。
```

## Final Checks

End with:

```text
【人工核验清单】
1. 需核验的法律依据：

3. 需补充或带原件的证据：
4. 需客户最终确认的事项：
5. 关键人物沟通和合规边界：
```

## Avoid

- Do not produce a long memo if the user asked for a hearing-night checklist.
- Do not repeat every simulation output. Synthesize.
- Do not hide fatal risks. The playbook exists to face them calmly.
