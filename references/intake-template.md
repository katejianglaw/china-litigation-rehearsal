# Intake Template

Use this reference when the user has not provided a structured case brief, or when the role simulation would be unreliable without more facts.

## Minimum Viable Intake

If the user wants a fast answer, ask only the smallest useful set of questions:

1. 案由、程序阶段、我方身份和核心诉讼请求是什么？
2. 法院或仲裁/诉讼衔接背景是什么？如不便说明，至少说明层级和地域类型。
3. 三个最关键事实分别是什么？每个事实对应哪些证据？
4. 对方最主要的抗辩是什么？
5. 是否存在仲裁条款、管辖争议、时效风险、举证期限、保全、鉴定、原件缺失等程序问题？
6. 你最想排练的是快速体检、标准预演、深度打磨、红队攻击、法官追问、关键人物陈述，还是败诉风险排查？

If the hearing is imminent, proceed with clear assumptions instead of over-asking.

## Confidentiality Intake Warning

Before receiving detailed facts, remind the user:

- 使用甲公司、乙公司、A先生等代号。
- 删除真实姓名、身份证号、联系方式、地址、账号、商业秘密和客户内部材料。
- 金额、日期、地点可以脱敏，但保留相对顺序、量级和法律意义。
- 涉外、数据出境、客户保密、监管调查、刑民交叉信息应单独标注，并谨慎输入。

## Scope Classification

If the case is outside ordinary mainland civil/commercial litigation, state the adjustment:

- 刑事、行政、执行异议、破产、家事、劳动仲裁衔接、涉外取证、跨境数据、保密或监管事项: continue only with explicit scope limits.
- If current legal authorities are necessary, mark `需检索核验` and do not rely on memory.
